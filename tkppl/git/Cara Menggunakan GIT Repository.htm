Cara Menggunakan GIT Repository

Dibuat : Nov 20th, 2013 09:26:59 Author : Danang Widiantoro
Jika anda sering bekerja dengan banyak file yang dikerjakan oleh lebih dari satu orang di masing-masing komputer mereka, maka ada baiknya anda menggunakan repository dimana repository ini akan mensinkronisasikan semua file2 anda dengan team anda di masing2 komputer tiap anggota team. Satu dari sekian jenis repository adalah GIT yang akan saya bahas di sini cara menggunakanya.

Download aplikasi GIT GUI

Ini hanya saran, tetapi jika anda sudah mengenal dengan baik atau setidaknya sudah familiar dengan perintah2 yang ada di GIT maka anda bisa menggukan terminal atau GIT-Bash untuk windows nya. Aplikasi GUI sangat di sarankan untuk mempermudah anda dalam menggunakan GIT, karena jika ada file yang terhapus dari sisi repository atau sekedar melihat apa aja yang telah dikerjakan oleh user lain maka aplikasi ini sangat memudahkan kita dalam memonitor perubahan2 file baik di sisi kita maupun di sisi user lain. Klik Di Sini Untuk mendapatkan master sourcetree lalu install ke OS anda. Jika anda menggunakan windows maka dalam proses penginstallan anda akan diminta untuk menginstall aplikasi tambahan seperti putty untuk ssh sebagai jembatan agar bisa terkoneksi dengan OS Linux, dimana sebagian besar server online menggunakan linux.

Pilih Aplikasi Repository yang diinginkan (Git / Mecurial)

Jika anda menggunakan windows maka anda harus menginstall salah satu repository yang akan anda gunakan, atau juga bisa dua duanya. Repository inilah yang akan menjadi referensi sourcetree dalam mensyncronisasikan file2 anda dengan file di server repository

Generate Key Pairs (sebagai pengganti Login)

Perlu diingat bahwa jika anda diminta memasukkan password dalam men-generate file keys (Key-Gen), password itu hanya berlaku saat anda akan menggunakan keygen tersebut (untuk menjaga2 jika laptop/komputer anda di pinjam oleh orang lain yang tidak bertanggunjawab dan meng-eksploitasi key-gen anda untuk masuk ke server2 dimana anda bekerja). Jadi lebih memudahkan jika anda tidak menentukan password apapun agar penggunaan key-gen anda lebih cepat krn tanpa memasukkan password.
Linux/Mac: anda cukup masuk ke terminal kemudian memasukkan perintah "ssh-keygen" yang akan men-generate file keys untuk pengganti login anda. Biasanya keygen private ada di ~/.ssh/id_rsa sedangkan public ada di ~/.ssh/id_rsa.pub
Windows: Jika anda sudah menginstall Putty/OpenSSH maka buka aplikasi tsb lalu cari menu untuk men-generate keys (Maaf saya kurang tahu caranya, monggo di gugling.. :-D), ingat2 lah posisi file keys yang telah tergenerate (akan ada dua file untuk public dan untuk private). Pada aplikasi sourcetree pilih menu "Tools / Add SSH Keys" lalu pilih private keys anda. Sedangkan public keys anda digunakan oleh server dimana GIT Repository diinstall. Klik di sini untuk detail penggunaan source tree di windows.

Keperluan General Perintah pada GIT:

  • Setup awal file repository: 
    git clone URL_ATAU_PATH_SERVER  PATH_TUJUAN_YANG_ADA_DI_KOMPUTER_LOKAL
     NB : ingat PATH_TUJUAN_YANG_ADA_DI_KOMPUTER_LOKAL haruslah kondisi kosong
  • Update file komputer lokal dari server 
    git pull
  • masukkan file lokal yang berubah ke dalam daftar commit 
    git add .
     NB: perhatikan ada tanda titik di akhir perintah
  • Commit / Simpan perubahan file di komputer lokal 
    git commit -m "Pesan notes untuk penyimpanan"
  • Kirim semua perubahan yang telah tersimpan (ter Commit) ke server 
    git push origin master
     NB:
    origin=file yang ada di komputer lokal kita
    master=nama branch atau cabang yang ada di server
    origin dan master tidak perlu anda rubah jika anda hanya bekerja pada satu branch/cabang.
    branch adalah cabang pengerjaan, semisal ada folder proposal yang di pegang oleh A dan B. A ingin merubah proposal untuk di tawarkan perusahaan djarum dan B ingin mempertahankan file2 proposal seperti adanya. maka A harus membuat branch baru agar perubahan yang dia lakukan tidak mempengaruhi file yang ada di komputer B maka di komputer si A master=djarum sedang si B master=master
  • Melihat file2 apa aja di komputer lokal yang telah diubah dan belum disimpan 
    git status
  • Ambil perubahan dari server dan pertahankan perubahan yang ada di komputer lokal (Kecuali jika file yang di server juga diubah oleh user lain maka file tersebut akan menggantikan file yg dilokal meskipun file lokal telah diubah) 
    git fetch --all
git reset --hard origin/master
  • Ambil perubahan dari server dan hapus semua perubahan yang ada di komputer lokal 
    git fetch origin master
git reset --hard FETCH_HEAD
git clean -df
  • Pindah URL atau path server repository 
    git remote set-url origin git://URL_YANG_BARU_DI_SINI

Ringkasan proses urutan kinerja GIT adalah

  1. Clone / setup file2 dari server repository (git clone url/path_server path_lokal)
  2. Ubah file2 yang diinginkan
  3. Masukkan file2 yang diubah ke waiting list (git add .)
  4. Simpan perubahan (git commit -m "Notes perubahan")
  5. Kirim ke server (git push origin master)
  6. Ambil update dari server (git pull)
